# Prhythm バンドメンバー As-Is Journey Map

対象スキル: `create-journey-map` mode:asis  
対象ペルソナ: 案件初期の情報をバンド資産に変えたい現場メンバー  
注意: 実ヒアリングではなく persona 文書を入力にしたドラフト。推測は `※推測` と明示する。

## ペルソナ確認

- 名前（仮名可）: 佐藤 亮 ※推測
- 属性: 案件初期の顧客ヒアリングや提案整理に入るバンドメンバー ※推測
- シナリオの範囲: 顧客ヒアリング前から、会議後整理、提案の芯を作るまで

## As-Is JM

### フェーズ 1: ヒアリング前に情報を集める

状況:  
ヒアリング前日の夕方。Slack のスレッド、Notion の過去メモ、営業からの短い共有を開いている。次の日の顧客ヒアリングに誰が何を聞くか、まだ明確ではない。※推測

発話 / 内心の声:  
「質問は作れるけど、結局何を確かめる会議なんだっけ」※推測

行動:  
過去資料をざっと読み、聞くべきことを個人メモに書き出す。バンド内で軽く共有するが、仮説の粒度は揃いきらない。※推測

感情:  
準備不足への不安と、時間をかけすぎられない焦りが混ざっている。※推測

タッチポイント:  
Slack、Notion、過去議事録、営業メモ、個人メモ。

課題:  
会議目的、既知情報、不明点、聞くべき質問が同じ場所に整理されていない。個人ごとの準備差が残る。※推測

### フェーズ 2: 顧客ヒアリングに入る

状況:  
顧客とのオンライン会議。バンドメンバーが複数名参加しているが、誰がどこまで深掘りするかはその場で調整している。※推測

発話 / 内心の声:  
「今の発言、あとで重要になりそうだけど、誰か拾えているかな」※推測

行動:  
顧客の発言をメモしながら、必要に応じて質問を追加する。時間が押すと、用意していた質問を飛ばす。※推測

感情:  
顧客の話から手がかりを得る期待と、聞き漏らしへの緊張がある。※推測

タッチポイント:  
オンライン会議、議事録、チャット、質問メモ。

課題:  
発言は残っても、その場で「論点」「仮説」「未確定事項」に分ける余裕がない。※推測

### フェーズ 3: 会議後に議事録を見返す

状況:  
会議後、次の予定までの短い時間で議事録を見返している。ログはあるが、どの発言が提案判断に効くのかはまだ整理されていない。※推測

発話 / 内心の声:  
「議事録はある。でも次に何を聞くべきか、これだけでは決めづらい」※推測

行動:  
重要そうな発言を拾い、Slack に共有する。必要なときだけ、別途まとめを作る。※推測

感情:  
会議直後の熱があるうちに整理したいが、作業が後回しになりそうな感覚がある。※推測

タッチポイント:  
議事録、Slack、Notion、個人メモ。

課題:  
議事録が「記録」で止まり、「次の判断に使える材料」へ変換されない。※推測

### フェーズ 4: 誰向けの提案かを考える

状況:  
顧客企業の中に複数の利用者、決裁者、影響者が見えてきた。提案書や次回会議に向けて、どの人を主対象にするかを考える。※推測

発話 / 内心の声:  
「全員向けにすると丸くなる。でも誰を中心にするかを勝手に決めるのも怖い」※推測

行動:  
議事録を見ながら、対象ユーザー候補や課題をメモする。チーム内で話すが、比較軸が揃わない。※推測

感情:  
提案を前に進めたい一方で、判断の根拠が薄いことへの不安がある。※推測

タッチポイント:  
議事録、提案メモ、ペルソナ候補、社内レビュー。

課題:  
対象候補の比較表や Counter-persona がないため、誰に最適化するかの議論が曖昧になる。※推測

### フェーズ 5: 提案の芯を言語化する

状況:  
社内共有や顧客への次回提案に向けて、Why / Who / What を短く説明する必要がある。※推測

発話 / 内心の声:  
「説明するたびに言い方が変わる。これで同じ提案として伝わるだろうか」※推測

行動:  
過去メモや議事録を読み返しながら、提案文や一行コンセプトを作る。必要に応じてメンバーにレビューを求める。※推測

感情:  
言語化できると前に進む感覚があるが、根拠が弱いと後で揺り戻しそうだと感じる。※推測

タッチポイント:  
提案書、Notion、Slack、コンセプト文。

課題:  
ヒアリング、ペルソナ、体験仮説、提案文が別々に作られ、つながりが見えにくい。※推測

## インサイト

1. 記録は残っているが、判断材料に変換するタイミングと型が弱い。  
2. チームで使う成果物と、個人が素早く前に進むためのメモが混在している。  
3. 誰向けに提案するかの判断が、比較可能な形で残りにくい。  
4. 上流の言葉が後続成果物へ渡る前に分断され、毎回再整理が起きる。  
5. Prhythm の価値は「全スキルを使うこと」ではなく、「次の判断へ渡す変換点」にある。

## HMW

1. どうすれば、ヒアリング前後の情報を短時間で判断材料に変換できるか？
2. どうすれば、個人の初動速度を落とさずに、後続メンバーが使える文脈を残せるか？
3. どうすれば、誰向けに提案するかをチームで比較しやすくできるか？
4. どうすれば、ヒアリング、ペルソナ、コンセプト、体験整理の間で言葉が切れないようにできるか？
5. どうすれば、Prhythm を全スキル連続利用ではなく、必要な変換点から軽く使い始められるか？
