# Prhythm — Vision & Concept

対象スキル: `product-vision-and-concept`  
入力: `docs/prhythm-band-persona-stage1.md`, `docs/prhythm-band-persona-stage2.md`

## One-line statement

> 案件初期の断片を、バンドが次の判断に使える共有材料へ変える。

## Concept

**Why** — バンド体制では、顧客ヒアリングや提案準備が個人の動きから始まることも、チームとして進むこともある。組織運用がまだ揺れている段階では、議事録や個人メモだけでは「なぜその判断になったか」が残りにくい。

**Who** — Prhythm は、案件初期の情報を後続メンバーも使える形にしたい現場メンバーのためのスキル集である。特に、ヒアリング前後の整理、対象ユーザーの比較、提案の芯の言語化を、軽く始めながら必要に応じてつなげたい人を想定する。

**What** — Prhythm は、ヒアリング準備、会議後分析、ペルソナ整理、ジャーニーマップ、コンセプト化、プロトタイプ前判断、実装前構造化までを、案件フェーズに応じた成果物として出す。すべてを順番に使わせるのではなく、今の判断に必要なスキルだけを使い、必要なら次へ渡せるようにする。

**How it's different** — 汎用AIの一問一答や、Notion の自由メモとは違い、Prhythm は「次に誰が何を判断するか」を成果物の形に含める。重い標準プロセスではなく、単発利用から 2〜3 スキル連携へ伸ばせることを重視する。

## Assumptions used

- Primary persona はまだ仮説段階である
- 実ユーザーが全スキルを連続利用したいとは限らない
- 最初の価値検証は、単発または 2〜3 スキル連携から始める
- 管理者向け標準化より、現場メンバーの案件前進を先に見る
