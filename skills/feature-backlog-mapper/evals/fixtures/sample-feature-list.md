# ReadAlong 機能一覧 / 簡易要件定義書

> 評価フィクスチャ。`feature-backlog-mapper` の Layer B シナリオ B-2 の入力として使う。
> Mode A 出力の見本。本物の出力は [proposal-template.md](../../references/proposal-template.md) に従う。

## スコープ

- **ビジョン**: 読んだ本のメモを、友達と気軽に共有できる軽い読書ノート。
- **対象ユーザー**: 月 2-5 冊読む社会人
- **このサイクルで含むもの**: D01 認証 / D02 読書ノート
- **このサイクルで含まないもの**: 友人招待・リアクション・通知（Phase 2）
- **入力ソース**: docs/usecase-map.md (UC 全 6 件 / 2 ドメイン)

## 機能カード

### F-D01-01 メール+パスワードでサインアップ

- **UC**: UC-D01-01
- **優先度**: Must — 根拠: ビジョン「読書ノートを共有」を成立させるにはアカウント必須
- **入力**: email (string), password (string ≥ 8 文字)
- **出力**: 新規ユーザーレコード, セッション Cookie, `/books` へリダイレクト
- **基本ルール**:
  - 同一 email の重複登録は不可
  - パスワードは bcrypt 等で保存
- **制約・前提**: メール確認フローは Phase 2
- **受入のスケッチ**: 新規 email → ダッシュボードへ / 既存 email → エラー / 短いパスワード → クライアントエラー

### F-D01-02 メール+パスワードでログイン

- **UC**: UC-D01-02
- **優先度**: Must — 根拠: 既存ユーザーが再開できないとビジョン成立せず
- **入力**: email, password
- **出力**: セッション Cookie, `/books` へリダイレクト
- **基本ルール**:
  - 失敗 5 回でアカウントロック 15 分
- **制約・前提**: パスワードリセットは Phase 2
- **受入のスケッチ**: 正しい credentials → /books / 誤り → エラー / 5 回失敗 → ロック挙動

### F-D01-03 ログアウト

- **UC**: UC-D01-03
- **優先度**: Must — 根拠: 共有端末利用時のセキュリティで必須
- **入力**: 認証済みリクエスト
- **出力**: セッション破棄, `/login` へリダイレクト
- **基本ルール**: Cookie を即時無効化
- **制約・前提**: なし
- **受入のスケッチ**: ログアウト → /login / 再アクセス → /login にリダイレクト

### F-D02-01 本を登録

- **UC**: UC-D02-01
- **優先度**: Must — 根拠: ノートが紐付く対象が無いと "ノートを書く" が成立しない
- **入力**: title (string), author (string), 任意 ISBN
- **出力**: book レコード, `/books/:id` へ遷移
- **基本ルール**:
  - 同一ユーザー内で同じ title + author は重複不可（警告表示）
- **制約・前提**: 書誌 API 連携は Phase 2
- **受入のスケッチ**: 必須フィールド入力 → 詳細画面 / 重複 → 警告 / 空欄 → クライアントエラー

### F-D02-02 ノートを作成

- **UC**: UC-D02-02
- **優先度**: Must — 根拠: ビジョン「メモを残す」のコア動作
- **入力**: book id, body (Markdown), visibility (default `private`)
- **出力**: note レコード, `/books/:id` で時系列に表示
- **基本ルール**:
  - body は 10,000 文字まで
  - 作成時刻で並ぶ（昇順 or 降順は UI 仕様で決定）
- **制約・前提**: リッチテキスト・画像添付は対象外
- **受入のスケッチ**: 入力 → 同画面に追記表示 / 空 body → クライアントエラー

### F-D02-03 ノートの可視性切替

- **UC**: UC-D02-03
- **優先度**: Should
- **入力**: note id, visibility (`private` / `friends`)
- **出力**: 更新済 note レコード
- **基本ルール**:
  - 切替は note 詳細画面の 1 ボタン
- **制約・前提**: 友人連携の機能（招待・閲覧）は Phase 2 のため、`friends` 設定は Phase 2 で見えるようになる
- **受入のスケッチ**: トグル → 状態変更 / 認可外 → 403
