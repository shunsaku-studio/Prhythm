# usecase-mapper

> Persona が固まった段階で、**Mermaid のユースケース図っぽい図** を主役にスケッチするスキル。

コードベースがまだ無くても使える前提で、Persona・ビジョン・要件メモ・議事録などから「誰が」「どんな機能を通じて」「何を達成したいか」を整理し、**Mermaid の全体俯瞰図と機能別ユースケース図を中心に** `docs/usecase-map.md` に出力します。

---

## 概要

`docs/usecase-map.md` に以下を出力します。

| 出力 | 内容 |
|---|---|
| **全体ユースケース図** | Mermaid で Persona / Actor と機能の関係を俯瞰する図 |
| **機能別ユースケース図** | Mermaid で特定機能に紐づくユースケースだけを描く図 |
| **Actor / Persona 一覧** | 誰が使うか、どんな状況か、何を達成したいか |
| **機能一覧** | どんな機能が必要か、その機能は何を可能にするか |
| **ユースケース一覧** | 図に乗せきれない補助情報 |

このスキルは**実装仕様書を作るものではない**。API、DB、画面パス、内部フローは扱わず、まず Mermaid 図で利用関係を掴むためのスケッチに絞る。

## 利用メリット

- Persona が決まったので、まず利用シーンを粗く出したい
- 一覧表より先に、アクターとユースケースの関係を Mermaid で見たい
- コードを書く前に、機能の抜け漏れを見たい
- ドメイン整理ではなく、「ユーザーに見える機能」の単位で考えたい
- 全体像と、特定機能だけの詳細を切り替えて見たい

## 利用シーン

### 1. 全体俯瞰モード

プロダクト全体をざっと見たいときに使います。

- 全体ユースケース図
- 機能別ユースケース図
- Actor / Persona 一覧
- 機能一覧
- 機能ごとの簡易ユースケース一覧

### 2. 機能別詳細モード

ある機能だけ深掘りしたいときに使います。

- 対象機能のユースケース図
- 対象機能の要約
- 対象機能のユースケース一覧
- 前提 / 未確定事項

## 使い方

Claude Code 上で以下のように依頼すると起動します。

```text
Persona をもとにユースケースをスケッチして
このプロダクトの機能一覧と全体ユースケース図を出して
予約機能だけユースケース図にして
```

入力ソースがある場合は一緒に指定できます。

```text
docs/prhythm-band-persona-stage2.md をもとに全体俯瞰で出して
このメモから「提案のたたき台作成」機能だけ詳細化して
```

## 前提条件

入力はコードなしでも成立する。主に以下を使う。

| 入力種別 | 例 | 主に拾う情報 |
|---|---|---|
| Persona / Segment 定義 | persona doc, interview synthesis | アクター、状況、目的 |
| Vision / Concept | vision doc, concept memo | 目指す価値、優先すべき方向性 |
| 要件メモ | 要求一覧、ユーザーストーリー | 機能候補、利用シーン |
| 議事録 / ヒアリング | MTG メモ、インタビュー | 生の文脈、困りごと、言い回し |
| 既存コード / 仕様書 | 任意 | 補助的な確認材料 |

## 出力例

最初に見るべきものは [example.md](example.md) の Mermaid 図です。一覧は補助情報です。

機能一覧:

| 機能ID | 機能名 | 何を可能にするか | 主な対象 Actor |
|---|---|---|---|
| F01 | 方向性を揃える | 提案や検証の前提を共有できる | バンドメンバー |
| F02 | ヒアリング準備を整える | 会話前に仮説と質問を揃えられる | 提案担当 |
| F03 | 次の検証論点を整理する | 何を次回確認すべきか決められる | 提案担当、PM |

ユースケース一覧:

| UC ID | 機能ID | ユースケース | 主アクター | 利用シーン | 期待結果 |
|---|---|---|---|---|---|
| UC-F01-01 | F01 | 提案の前提を共有できる | バンドメンバー | 初回提案の前 | 認識のズレが減る |
| UC-F02-01 | F02 | ヒアリング前に仮説を揃えられる | 提案担当 | 顧客訪問の前日 | 質問が散らからない |

より具体的なテスト出力と Mermaid 図は [example.md](example.md) を参照。

## 注意事項

- API 一覧を出す
- 画面一覧を出す
- DB / スキーマを推定する
- 実装済み / 未実装を断定する
- 内部フロー図を描く

これらは後続の設計スキルで扱う。`usecase-mapper` は、**機能と利用シーンの骨組み** に集中する。

## 構成

```text
usecase-mapper/
├── SKILL.md     # エージェントが読む実行手順
├── README.md    # 人間向けの概要
├── example.md   # Persona 入力を使った実出力例
```
