# hearing-prep

> ヒアリング前準備と当日の進行カードを作るスキル。

## 概要

営業初回ヒアリングに開発メンバーや PM が同席する前提で、会議中に迷わないための準備物を作る。

## 利用メリット

- **会議前に迷いが減る** - 目的、進行、質問、メモ観点が一式そろう
- **営業的な初回ヒアリングに開発が入りやすい** - 何を聞いて何をまだ決めないかが明確になる
- **質問の再利用が効く** - `questions.md` の静的質問集を毎回の会議に合わせて抜粋できる
- **分析しすぎを防げる** - ボトルネック断定や解決策断定を意図的に避ける

## 利用シーン

- **初回ヒアリング前に準備したいとき** - 会議目的と不明点を整理する
- **会議中に見る進行カードがほしいとき** - 会議の流れとその場で聞く質問を出す
- **同じテーマのヒアリングが複数回続くとき** - `questions.md` をデータソースにして質問候補を使い回す
- **営業メモしかない状態で臨むとき** - 既知情報と不明点を切り分ける

## 使い方

```text
/hearing-prep
顧客名: ○○
業種: 製造業
相手の役職: 品質管理部長
今回の相談内容: 月次レポート作成の負荷が高い
会議の目的: 現状把握と次回検討論点の整理
既に分かっていること: 月3日かかる。AI に関心あり
まだ分かっていないこと: どの工程が重いか、読者、制約
時間: 45分
参加者: 顧客2名、営業1名、PM1名
```

## 構成

```text
hearing-prep/
├── README.md
├── SKILL.md
├── example.md
└── questions.md
```

## 前提条件

- 会議前の入力として、顧客情報や営業メモが最低限あること
- 質問ソースとして [questions.md](questions.md) を読むこと
- 会議前準備であり、事後分析ではないこと

## 注意事項

- このスキルでは、ボトルネック分析をしない
- `questions.md` の全質問をそのまま出さず、会議目的に合わせて選定する
- AI 化や自動化を前提に誘導しない
