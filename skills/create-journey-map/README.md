# create-journey-map

> ヒアリング情報から **ジャーニーマップ（JM）** を台本形式で生成し、課題発見からHMW・コアシーン候補までつなぐエージェントスキル。

## 概要

ペルソナのヒアリングメモやインタビュー記録をもとに、ユーザー体験を時系列で可視化するジャーニーマップを生成します。現状（As-Is）と理想（To-Be）の2モードを持ち、As-Isでは課題からインサイト・HMW（How Might We）への変換まで、To-Beでは対比サマリーとプロトタイプ検証用のコアシーン候補まで導きます。感情を数値ではなく言葉で描く「台本形式」で、ありありとした体験の解像度を重視します。

## 利用メリット

- **現状の体験が物語として見える** — 場所・発話・感情まで描かれ、チームが同じシーンを思い浮かべられる
- **課題が「構造的な問い」に変わる** — 表面的な不満の列挙で終わらず、インサイト → HMW という次の打ち手の起点が揃う
- **ウィッシュリスト的なTo-Beを避けられる** — HMWの合意を挟むことで、現状の課題を起点にした現実的な理想像になる
- **次に何を試すかが絞れる** — To-Beから検証すべきコアシーン候補が提示され、プロトタイプ設計につながる

## 利用シーン

- **ヒアリングは終えたが、体験全体を捉えきれていないとき** — 断片的なメモを時系列の物語に整理したい
- **課題は感じているが、何に取り組むべきか曖昧なとき** — 構造的なインサイトとHMWで論点を立てたい
- **チームでHMWの優先度を決めた後** — 合意した方向性をもとに理想の体験を描きたい
- **プロトタイプで何を検証するか迷っているとき** — 体験の中から優先して試すべきシーンを見極めたい

依頼の例: 「このヒアリングメモからAs-Is JMを作って」「合意したHMWをもとにTo-Be JMを作って」

## 使い方

```
このヒアリングメモとペルソナ情報からAs-Is JMを作って
```

```
このAs-Is JMと合意したHMW（最優先：〇〇）をもとにTo-Be JMを作って
```

モードを指定しない場合は mode:asis（As-Is生成）で実行します。

## 構成

```
create-journey-map/
├── README.md
├── SKILL.md                    # モード判定・共通ルール（モデルが読む本体）
└── references/
    ├── mode-asis.md            # As-Is JM → インサイト → HMW の詳細手順
    └── mode-tobe.md            # To-Be JM → 対比サマリー → コアシーン候補の詳細手順
```

## 前提条件

- Claude Code（プラグイン `prhythm` の一部として配布）
- 最低限のインプット：ペルソナの基本属性と、現状の体験の概要（ヒアリングメモ・議事録など）
- mode:tobe を使う場合：As-Is JMの全文と、チームが合意した最優先HMW

## 注意事項

- ヒアリング情報が不足する箇所は推測で補い、`※推測` と明示します。必須情報が欠けている場合は生成せず質問します
- mode:asis は HMW 提示後に必ず停止します。To-Beへは、チームがHMW優先度を合意してから mode:tobe で進みます
- 感情は数値スコア（1〜5）では表現しません（定性的なJMでは言葉で描くことを優先）
- 定量的なセグメンテーションや統計的分析は対象外です
