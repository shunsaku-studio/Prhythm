# competitive-research

## 概要

新規プロダクト・機能を考えるときに、**競合・参考サービスを調査**するスキル。サービス一覧の羅列ではなく、**自プロダクトの設計判断に使えるインサイト**を出すことを目的とする。

- **入力:** 解決したいユーザー課題、スコープ（地域・プラットフォーム・セグメント）、調査の焦点（UX / ビジネスモデル / 機能 / ポジショニング）
- **出力:** Phase 1 は 15–20 件のカタログ（広域スキャン時は最大 30）と深掘り候補の推薦、Phase 2 は個別プロファイル・横断比較・自プロダクト向け Recommendations
- **スコープ外:** 単一サービスの機能解説のみ、プロダクト設計意図のない市場規模・業界分析

## 利用メリット

- **設計判断の材料が揃う** — 競合の UX・料金・オンボーディング・ユーザーの不満から、「借りる／避ける」が具体化する
- **市場の全体像を短時間で掴める** — 広くスキャンしたあと、深掘りすべき 5–8 件が推薦されるので、どこから読むか迷わない
- **直接競合以外からも学べる** — 同じ操作パターンや別業界のビジネスモデルから、自プロダクトに転用できる発想が得られる
- **ポジショニングの空白が見える** — 2 軸マップで混雑地帯と未開拓地帯が整理され、差別化の方向性を議論しやすい
- **毎回同じ粒度で議論できる** — 広域スキャンと深掘りの切り口が揃い、チームで結果を共有しやすい
- **調査が会話で続く** — 各フェーズ末にギャップと次の一手が示され、一発出力で終わらない

## 利用シーン

- **アイデアはあるが、同じ課題を誰がどう解いているか把握していないとき** — 参考サービスの全体像と深掘り候補が欲しい
- **カテゴリ名だけ分かっているとき** — 誰の課題かを先に揃えてから調査に入れる
- **調査の途中でターゲットや焦点を絞りたいとき** — 絞り込み後も relevant なサービスだけ深掘りできる
- **競合は知っているが、自プロダクトに何を学ぶか絞り込みたいとき** — 推薦リストから 5–8 件を選んで詳細分析したい
- **画面やオンボーディングを設計する前に、他社のやり方を知りたいとき** — UX パターンやユーザー sentiment を材料にしたい
- **料金・機能・ターゲットの置き方がまだ曖昧なとき** — ポジショニングマップで自コンセプトの位置を議論したい
- **スキーマ設計や OOUI のたたき台に入る前に、参考プロダクトを整理したいとき** — 次の設計フェーズに渡せる調査結果が欲しい

依頼の例: 「予約 SaaS の競合調査をして」「経費精算アプリの参考サービスを探して、UX パターン中心に整理して」

## 使い方

**Phase 0（フレーミング）** — 課題・スコープ・焦点を確認。カテゴリのみの入力時は最小 1 問（詳細は `references/phase0-template.md`）

**Phase 1（Broad Scan）** — 15–20 件のカタログと `Recommended for Deep Dive` を出力（テンプレ: `references/phase1-template.md`）:

```
競合調査して。SMB 向けの AI 経費精算ツールを作りたい
What services exist in the booking SaaS space?
参考サービスを探して、オンボーディングの UX パターンが知りたい
```

**Phase 2（Deep Dive）** — Phase 1 の候補またはユーザー指定のショートリストを深掘り（テンプレ: `references/phase2-template.md`）。フレーム変更時は Selection Rationale 表を先に出す:

```
Phase 1 の推薦 5 件を深掘りして
X, Y, Z を詳しく分析して。ポジショニングマップも欲しい
共働き向け・女性キャリア支援に絞って深掘りして
```

**品質確認** — 送信前に `references/output-checklist.md` を満たしているか確認

## 構成

```
competitive-research/
├── README.md
├── SKILL.md
└── references/
    ├── phase0-template.md      # 調査フレーム・カテゴリのみ入力時の 1 問
    ├── phase1-template.md      # Broad Scan 出力構造
    ├── phase2-template.md      # Deep Dive 出力構造（3+3 sentiment 表）
    ├── shortlist-selection.md  # 5–8 件選定・フレーム変更時の再選定
    ├── sentiment-sources.md    # 口コミソース優先順位
    └── output-checklist.md     # フェーズ別品質ゲート
```

## 前提条件

- **Web 検索** — 公式サイト、App Store / G2、Product Hunt 等（sentiment-sources 参照）
- **日英検索** — グローバル・国内双方の参考サービスを拾う想定（必要に応じて他言語も）
- スキル配置: Prhythm の `skills/` または `~/.cursor/skills/`

## 注意事項

- **単一サービスの機能解説だけ**が目的のときは使わない — その場で回答するか、Phase 2 Layer 1 の 1 件プロファイルに限定する
- **市場規模・業界レポートのみ**でプロダクト設計に直結しない調査は対象外
- Phase 1 は一覧の深さを揃えすぎない — スキャン用の短いエントリ + 推薦セクションが本体
- Phase 2 はプロファイルだけで終わらせない — Cross-Comparison と Insights for Your Product まで出す
- 料金・規模は `Verified` / `二次情報` / `要確認` を付ける — 比較記事だけの数字を確定事実にしない
- 各フェーズ後は調査のギャップと次ステップを添える
