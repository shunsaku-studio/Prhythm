# Output Checklist

Run mentally before sending each phase. All items should pass.

---

## Phase 0

- [ ] Problem is **pain**, not category label alone
- [ ] If user gave category only → **asked minimum question** OR stated `(前提)` assumptions explicitly
- [ ] Scope and focus recorded in frame table

## Phase 1

- [ ] 15–20 entries (≤30 only if user requested wider scan)
- [ ] Every entry has: What, Relevance, Why listed, Scale signal
- [ ] **Verified / 二次情報 / 要確認** on uncertain facts
- [ ] Mix includes non-direct references (UX or BM)
- [ ] **Recommended for Deep Dive** section present with 5–8 picks + table
- [ ] Gaps & suggested next steps at end
- [ ] Japanese + English search when applicable

## Phase 2

- [ ] **Selection Rationale** table at top (especially after frame change)
- [ ] 5–8 services; 2–4 searches each
- [ ] Each profile: Basics, VP, Product, UX, **sentiment 3+3 table**
- [ ] Sentiment from priority sources ([sentiment-sources.md](sentiment-sources.md)), not copy alone
- [ ] Cross-Comparison: matrix + positioning map with **product-relevant axes**
- [ ] Insights: patterns, differentiation, opportunities, **Borrow / Avoid**
- [ ] Synthesis not summary ("most assume Z, but reviews show…")
- [ ] Gaps & suggested next steps at end

## Anti-patterns (fail if true)

- [ ] Directory dump without Why listed
- [ ] Phase 2 profiles only — no Cross-Comparison or Insights
- [ ] Generic axes: "simple vs feature-rich" without user frame tie-in
- [ ] Phase 2 started without Selection Rationale when frame shifted
