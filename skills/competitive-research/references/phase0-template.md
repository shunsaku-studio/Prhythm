# Phase 0 Output Template

Use at the start of every research run. Copy the structure below; fill or mark `（前提）` / `（要確認）`.

---

## Phase 0: Research Frame

| Field | Content |
|-------|---------|
| **Problem (pain)** | {Who has what pain — not category name} |
| **Target user** | {Segment, e.g. dual-income career women} |
| **Scope** | {Geography, platform, B2C/B2B} |
| **Focus** | {UX / business model / features / positioning — pick 1–2 primary} |
| **Assumptions** | {List anything inferred; flag if user did not confirm} |

### When to ask vs assume

| User input | Action |
|------------|--------|
| Category only ("家事代行", "booking SaaS") | **Ask one question** before Phase 1: "Who is the primary user and what pain are you solving?" Offer 2–3 pain/segment options. |
| Category + segment or pain | State frame; proceed to Phase 1 |
| Full product brief in conversation | State frame as assumptions; skip questions |

### Minimum question (category-only input)

```
調査の前に1点だけ確認させてください。

- **誰の、どんな課題**を解くプロダクト想定ですか？
  （例: 共働き世帯の家事負担 / SMB の経費処理 / 〜）

カテゴリだけ分かっている場合は、こちらで前提を置いて Phase 1 に進みます。
```

If the user does not answer, proceed with stated assumptions and label them `(前提)`.
