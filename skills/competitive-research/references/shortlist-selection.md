# Shortlist Selection

How to pick 5–8 services for Phase 2 and document the choice.

---

## Triggers for Phase 2

- User picks from Phase 1 Recommended list
- User names services: "X, Y, Z を深掘り"
- User **refines the research frame** (new segment, pain, or focus) — re-select shortlist; do not reuse Phase 1 picks blindly

---

## When the user changes the frame

Example: Phase 1 was generic "家事代行" → user adds "共働き向け・女性の社会進出".

1. **Update Phase 0** table (problem, target, focus)
2. **Re-score Phase 1 catalog** against the new frame — do not restart Phase 1 unless coverage is clearly wrong
3. Output **Selection Rationale** table (required at top of Phase 2):

| Service | From Phase 1? | Why selected for this frame |
|---------|---------------|----------------------------|
| タスカジ | Yes | Mission-aligned; pull-model reference |
| CaSy | Yes | App UX + dual-income default choice |
| … | | |

4. Explain **dropped** high-profile services if non-obvious (e.g. "おそうじ本舗 — HC-only, not daily housekeeping for career segment")

---

## Selection rules

| Relevance mix (5–8 total) | Guideline |
|---------------------------|-----------|
| Direct competitors | 3–5 |
| UX or BM reference | 1–2 |
| Adjacent / channel (e.g. B2B welfare) | 0–1 if frame relevant |

- Every pick must answer: **"What design decision does this help the user make?"**
- Prefer services with **verifiable sentiment** (app store, official reviews)
- If a Phase 1 entry was `要確認` or unverified, either verify in Phase 2 or drop with note

---

## User did not pick — agent selects

When proceeding without explicit picks:

1. Take Phase 1 **Recommended for Deep Dive** as default
2. Re-filter through updated Phase 0 frame
3. Show Selection Rationale table before profiles
4. Offer: "この6社で進めた。入替えたいサービスがあれば指定してください"
