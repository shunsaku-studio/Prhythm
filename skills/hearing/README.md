# hearing

> 顧客ヒアリング支援の入口スキル。依頼内容を `hearing-prep` と `hearing-analysis` に振り分ける。

## 概要

営業初回ヒアリングに開発メンバーや PM が同席するケースを前提に、会議前の準備と会議後の分析を分けて扱う。`hearing` 自体は薄い入口で、入力内容を見て `hearing-prep` と `hearing-analysis` のどちらを使うか決める。

## 利用メリット

- **入口が 1 つで迷いにくい** - 事前準備と事後分析を同じ名前空間で扱える
- **責務が混ざらない** - 会議前に分析しすぎず、会議後に初めてボトルネック仮説へ進める
- **既存資産を活かせる** - 旧 `hearing` の仮説更新資産を `hearing-analysis` に寄せたまま、初回ヒアリング準備も別スキルとして扱える

## 利用シーン

- **初回ヒアリング前に何を聞くか整理したいとき** - `hearing-prep` に振り分ける
- **会議中に使う進行カードがほしいとき** - `hearing-prep` に振り分ける
- **会議後にボトルネック仮説と次回確認事項を整理したいとき** - `hearing-analysis` に振り分ける
- **どちらを使うべきか曖昧なとき** - まず `hearing` を呼べばよい

| スキル | 使う場面 | 主な出力 |
|---|---|---|
| `hearing-prep` | 会議前に準備したい / 会議中の台本がほしい | 会議目的、相手理解、既知情報、不明点、進行カード、選定質問、SPIN ベースのカテゴリ別質問集、メモ観点 |
| `hearing-analysis` | 会議後に分析したい / 次回の擦り合わせ論点がほしい | 目的整理、現状整理、表面課題、ボトルネック仮説、根拠、解決方向、初期スコープ、次回確認事項、擦り合わせ質問集 |

## 使い方

まず `hearing` を入口として呼んでもよいし、明示的に各サブスキルを呼んでもよい。

```text
/hearing 来週の初回ヒアリング準備をしたい。営業メモはこれ: ...
```

```text
/hearing-prep 顧客名: ...
今回の相談内容: ...
会議前準備と当日の進行カードを作って
```

```text
/hearing-analysis ヒアリング結果: ...
こちらの仮説: ...
次回の擦り合わせ質問を出して
```

## 前提条件

- Claude Code 上で使う想定
- 営業メモ、顧客発言、議事録、社内仮説などの断片情報が最低限あること
- Web 検索は不要。入力された情報だけで扱う

## 注意事項

- `hearing-prep` は分析しない。会議前の準備だけを扱う
- `hearing-analysis` は準備カードを主目的にしない。会議後の整理と仮説更新に集中する
- AI 化や自動化を前提にした結論へ飛ばない

## 関連スキル

| スキル | 関係 |
|--------|------|
| hearing-prep | 前段。会議前準備と進行カードを作る |
| hearing-analysis | 後段。会議後の整理と擦り合わせ質問を作る |
| defining-personas-and-segments | 補完。誰向けに作るかの比較が主題になったらこちらを使う |
| product-vision-and-concept | 補完。ヒアリング材料をもとに構想整理へ進む段階で使う |
