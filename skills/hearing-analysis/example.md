# hearing-analysis 出力例

## 入力

```text
ヒアリング前の仮説: レポート文章生成が重いのではないか
ヒアリング結果:
- 月次レポート作成に3日かかる
- データ収集に2日、考察に1日
- 文章化自体は30分程度
- Excel フォーマットが部署ごとに少し違う
- 最終承認は部長が行う
```

## 出力イメージ

### 4. ボトルネック仮説

- 本当に詰まっていそうな場所: レポート文章生成ではなく、部署ごとに形式が揺れる Excel からのデータ収集と突合
- 詰まり方の種類: 作業量 + データ構造の不安定さ
- まだ未確認の前提: 部署差分が毎月固定か、担当者判断で変わるか

### 5. 根拠

- 発言ベースの根拠: 「データ収集に2日」「文章化は30分程度」
- 観測ベースの根拠: Excel フォーマットが部署ごとに異なる
- こちらの推測: 文書生成より先に標準化または集計整流化の価値が高い可能性

### 9. 擦り合わせ質問集

#### ボトルネック仮説の擦り合わせ

1. 部署ごとの Excel 差分は、列構成の違いですか、それとも入力ルールの違いですか？
   - どのズレを確認するためか: こちらが見ている「データ構造問題」が実態に合っているか
   - 回答で何を判断するか: 標準化が先か、収集フロー変更が先か
2. 収集に 2 日かかるのは、ファイルを集める時間と整形する時間のどちらが大きいですか？
   - どのズレを確認するためか: 作業量ボトルネックか、整形ボトルネックか
   - 回答で何を判断するか: 初期スコープの切り方
