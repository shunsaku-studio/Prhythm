# 中小工務店向け 現場進捗共有ツール 検討ブリーフ

## 1. 課題

中小工務店(社員5〜30人規模)では、現場ごとの進捗共有が紙の日報・電話・LINE で行われている。
社長や工事部長が現場の状況を把握するのに時間がかかり、トラブル発生時の初動が遅れる。
事務員側は、現場からの紙日報を Excel に転記する作業が日々発生している。

## 2. ターゲットの現時点の想定

- 中小規模の工務店(年商3〜30億)
- 複数現場を同時に持つ
- スマホは普及しているが、PC 利用率は限定的
- 現場監督 / 職人 / 事務員 / 社長 で関与の仕方が違う

## 3. 既存の代替手段

- 紙日報(大半の会社が運用)
- LINE グループ(現場ごとに作っている)
- 大手向け建設業務システム(機能・価格ともに過剰で中小は導入しない)
- スプレッドシート(若手社長が独自運用)

## 4. 関係者ヒアリング (各1名)

### 現場監督 (40代男性、3現場兼任)

- 日報を書くのが面倒、書く時間が取れない
- 写真を撮るのは慣れた、文字を打つのは苦手
- 「現場で書ける、声で済むとありがたい」
- 一方で「全部報告しろと言われると現場が回らない」
- LINE は使うが、業務として残らないのが不満

### 職人 (50代男性、施工担当)

- 報告はリーダーがやることで、自分は関与しない
- LINE は使うが、業務連絡で来ると返信が面倒
- 「写真だけ送るのは別に良い」
- スマホアプリの新規インストールは抵抗あり

### 社長 (50代男性、社員20名規模)

- 現場の進捗が見えなくて困っている
- 月に1度、現場を回って把握している
- 「リアルタイムで見えるならありがたいが、現場が疲弊するなら本末転倒」
- 価格は月3万円までなら検討する

### 事務員 (30代女性、入社5年目)

- 現場からの紙日報を Excel に転記している
- 「この作業がなくなるなら助かる」
- 「ただ、現場からの情報が遅れたら逆に困る」
- 現場監督からの日報は曜日にムラがあり、まとめて来る日もある

## 5. 定量メモ(ざっくり)

- 想定対象社数: 全国で約4〜5万社(社員5〜30人規模)
- 月額単価想定: 1社あたり 1〜3万円程度
- 1社あたりの現場数: 5〜20
- 1現場あたりの工期: 2週間〜6ヶ月

## 6. まだ決まっていないこと

- 現場監督 / 社長 / 事務員 のうち、誰の課題を最優先にするか
- 音声入力 / 写真ベース / テンプレ手入力 のどれを軸に据えるか
- 既存 LINE グループとの併用前提か、置換を目指すか
- まず何社で PoC するか
- 値付け(現場ごと課金 / 会社ごと固定 / 人数連動)

## 7. 仮説

- 現場監督の入力負荷を下げないと、何を作っても続かない
- 社長は意思決定者だが、エンドユーザーは現場監督
- 写真ベースの方が、テキストより現場の抵抗が少ない可能性
- 大手システムの「全部入り」が嫌われる理由は、機能数より導入・教育コスト
- 一気に全現場展開ではなく、1〜2現場で試せる導入が刺さりそう
